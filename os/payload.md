# Payload（系统负载与可观测性指标）

## 服务器负载的衡量标准

> 评估服务器性能, 稳定性和资源利用率的关键指标.

### CPU 相关指标

- **CPU 使用率 (%)**
  - 用户态 (`%us`), 内核态 (`%sy`), 空闲 (`%id`), 等待 I/O (`%wa`)
  - **警戒线**: 持续 >70% 可能需优化; >90% 需紧急处理
- **负载平均值 (Load Average)**
  - `1分钟/5分钟/15分钟` 的平均活跃进程数 (如 0.5, 1.2, 0.8)
  - **多核换算**: 若 CPU 有 4 核, 负载 4.0 = 满负荷
- **上下文切换 (Context Switches)**
  - 频繁切换 (如 >10 万/秒) 可能因进程过多或锁竞争

### 内存相关指标

- **内存使用率 (%)**
  - 包括已用 (`used`), 缓存 (`cached`), 缓冲 (`buffers`), 空闲 (`free`)
  - **注意**: Linux 会利用缓存提升性能, 需结合 `free -h` 分析
- **Swap 使用率**
  - Swap 频繁读写 (`si/so`) 说明物理内存不足, 性能急剧下降
- **OOM (Out of Memory) 事件**
  - 内核强制终止进程时触发, 需紧急扩容或优化内存泄漏

### 磁盘 I/O 指标

- **磁盘使用率 (%)**
  - 通过 `df -h` 查看分区占用, >80% 需清理或扩容
- **I/O 等待 (`%wa`**
  - CPU 等待磁盘响应的比例, >30% 可能磁盘瓶颈
- **读写吞吐量 (MB/s)**
  - 通过 `iostat` 监控 `r/s` (读), `w/s` (写)
- **磁盘队列长度**
  - 平均队列长度 (`avgqu-sz`) >2 可能磁盘过载

### 网络相关指标

- **带宽使用率 (%)**
  - 入向/出向流量占带宽上限的比例 (如 1Gbps 网络持续 >800Mbps 需关注)
- **连接数 (TCP/UDP)**
  - 通过 `netstat` 或 `ss` 查看活跃连接数, 尤其注意 `TIME_WAIT` 堆积
- **丢包率/错误率**
  - `ifconfig` 中的 `dropped` 或 `errors` 字段, >0.1% 需排查
- **延迟 (Latency)**
  - `ping` 延迟或应用层延迟 (如 API 响应时间 >500ms)

### 应用层指标

- **QPS (Queries Per Second)**
  - 每秒请求数, 如 Web 服务器 `Nginx` 的 `requests/sec`
- **响应时间 (Response Time)**
  - P50/P95/P99 分位值 (如 P99 <200ms)
- **错误率 (%)**
  - HTTP 5xx 错误比例, >1% 需告警
- **线程池/连接池状态**
  - 数据库连接池活跃连接数, 线程池队列堆积

### 系统级综合指标

- **系统吞吐量 (Throughput)**
  - 单位时间内处理的请求总量 (如 1000 事务/秒)
- **资源饱和度 (Saturation)**
  - 资源排队情况 (如 CPU 运行队列长度 `vmstat 1` 的 `r` 列)
- **UPtime (系统运行时间)**
  - 长期高负载可能导致服务劣化, 需定期重启

### 容器/云环境特有指标

- **容器 CPU/Memory Limit**
  - Kubernetes 中容器的 `limits/requests` 使用率
- **自动扩缩容触发条件**
  - 如 CPU >60% 持续 5 分钟触发 HPA
- **云服务配额**
  - 如 AWS EC2 的 vCPU 配额, EBS 带宽限制

### 常用监控工具

| 工具         | 功能                    |
| ------------ | ----------------------- |
| `top/htop`   | 实时 CPU/内存监控       |
| `vmstat`     | 系统级 CPU/内存/IO 统计 |
| `iostat`     | 磁盘 I/O 详细分析       |
| `nload`      | 网络流量监控            |
| `Prometheus` | 时序数据库 + 告警       |
| `Grafana`    | 可视化面板              |
| `ELK`        | 日志分析 (结合负载指标) |

### 负载高低判断参考

- **低负载**: CPU <30%, 内存 <50%, Load < CPU 核数
- **正常负载**: CPU 30%~70%, Load ≈ CPU 核数, 无持续瓶颈
- **高负载**: CPU >70%, Load > 2\*CPU 核数, Swap 使用或 IO 等待高
- **危险负载**: CPU >90%, Load 飙升, 服务超时或崩溃

### 优化建议

- **纵向扩容**: 升级 CPU/内存/磁盘
- **横向扩容**: 负载均衡 + 多实例部署

- **代码优化**: 减少锁竞争, 优化 SQL, 缓存热点数据
- **限流降级**: 熔断非核心服务 (如 Sentinel、Hystrix)

通过综合这些指标, 可以全面评估服务器负载并制定针对性策略
