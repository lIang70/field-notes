# C++14

## TL;DR

- C++14 可以理解为“把 C++11 的很多痛点打磨顺了”：更好用的 lambda、更容易写泛型代码、更自然的返回类型推导。

## 相关链接

- `c++11.md`（移动语义、智能指针、并发基础）
- `c++17.md`（结构化绑定、`if constexpr`、`optional/variant`）

## 1) 函数返回类型推导（`auto`）

- 允许直接写：

```c++
auto add(int a, int b) { return a + b; }
```

> 复杂模板场景下，返回类型推导能显著减少 `decltype(...)` 的样板代码。

## 2) 泛型 lambda（Generic Lambda）

- lambda 参数可以用 `auto`：

```c++
auto eq = [](const auto& a, const auto& b) { return a == b; };
```

## 3) `std::make_unique`

- C++14 正式提供 `std::make_unique`。
- 工程上：优先用 `make_unique/make_shared` 来避免裸 `new` 暴露在业务代码里。

## 4) `constexpr` 放宽

- C++14 起 `constexpr` 函数更接近“可在编译期执行的普通函数”，能写更复杂的逻辑。
