# C++17

## TL;DR

- C++17 的工程价值：**把很多“模板分支/错误处理/字符串视图/类型表达”变得更直接**，并提供一批更安全的标准类型（`optional/variant/any`）。

## 相关链接

- `c++11.md`（智能指针/并发基础）
- `../keywords.md`（cast、RTTI、UB）

## 1) 结构化绑定（Structured Bindings）

```c++
auto [x, y] = std::pair<int,int>{1, 2};
```

工程要点：

- 读写性更好，但注意绑定到引用时的生命周期（尤其是绑定到临时对象）。

## 2) `if constexpr`

- 模板分支在编译期裁剪，常用于替代复杂的 SFINAE：

```c++
template <class T>
void f(T v) {
    if constexpr (std::is_integral_v<T>) {
        // ...
    } else {
        // ...
    }
}
```

## 3) `std::optional` / `std::variant` / `std::any`

- `optional<T>`：表达“可能没有值”，替代“返回值 + out 参数/魔法值”。
- `variant<A,B,...>`：类型安全的联合体，替代 C 风格 `union` + tag。
- `any`：类型擦除容器，谨慎使用（可读性/性能/可维护性成本更高）。

工程建议：

- 如果你在写大量 `dynamic_cast` 来做类型分发，优先评估 `variant + visit`。

## 4) `std::string_view`

- 轻量“字符串切片视图”，避免无谓拷贝。
- 工程坑点：它**不拥有底层内存**，必须保证被引用的字符序列在 `string_view` 生命周期内有效。

## 5) `std::filesystem`

- 标准化的文件系统路径/遍历/操作 API（跨平台工程更稳）。
