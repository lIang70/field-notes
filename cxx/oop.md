## 常见坑（工程视角）

- **对象切片（Object Slicing）**：`Base b = derived;` 会丢掉派生类部分；多态应通过 `Base&/Base*` 使用。
- **基类析构不虚**：通过基类指针 `delete` 派生对象时会 UB/泄漏资源；基类作为多态基类时应提供 `virtual ~Base() = default;`。
- **隐藏 vs 重写**：签名不一致时是 name hiding，不是 override；建议一律用 `override` 让编译器帮你检查。

## 三大特性: 继承, 封装和多态

### 类的继承

- 类与类之间的关系
  - HAS-A: 用以描述一个类由多个部件类构成, 实现 HAS-A 关系用类的成员属性表示，即一个类的成员属性是另一个已经定义好的类
  - USE-A: 一个类使用另一个类, 通过类之间的成员函数相互联系, 定义友元或者通过传递参数的方式来实现
  - IS-A: 继承关系, 关系具有传递性
- 所谓的继承就是一个类继承了另一个类的**属性和方法**, 这个新的类包含了上一个类的属性和方法, 被称为子类或者派生类, 被继承的类称为父类或者基类
- 特点: 子类拥有父类的所有属性和方法, 子类可以拥有父类没有的属性和方法, 子类对象可以当做父类对象使用
- 访问控制: public, protected, private

### 类的封装

- 数据和代码捆绑在一起, 避免外界干扰和不确定性访问
- 也就是把客观事物封装成抽象的类, 并且类可以把自己的数据和方法只让可信的类或者对象操作, 对不可信的进行信息隐藏; 例如: 将公共的数据或方法使用 `public` 修饰, 而不希望被访问的数据或方法采用 `private` 修饰

### 类的多态

- 同一事物表现出不同事物的能力, 即向不同对象发送同一消息, 不同的对象在接收时会产生不同的行为**(重载实现编译时多态, 虚函数实现运行时多态)**
- 多态性是允许你将父对象设置成为和一个或更多的他的子对象相等的技术, 赋值之后, 父对象就可以根据当前赋值给它的子对象的特性以不同的方式运作. 简单一句话: 允许将子类类型的指针赋值给父类类型的指针
- 实现多态有二种方式: 覆盖 (override), 重载 (overload)
  - 覆盖: 指子类重新定义父类的虚函数的做法
  - 重载: 是指允许存在多个同名函数, 而这些函数的参数表不同 (或许参数个数不同, 或许参数类型不同, 或许两者都不同). 例如: 基类是一个抽象对象: 人;那教师, 运动员也是人, 而使用这个抽象对象既可以表示教师, 也可以表示运动员

#### `override`

1. 基本条件
   1. **基类函数是 `virtual`** (非虚函数的重定义叫 **隐藏 (Hiding)**, 不是重写)

   2. **函数签名相同**:
      - 函数名, 参数列表 (类型, 顺序, 数量) 必须完全一致

      - 返回类型通常需相同, 但允许 **协变返回类型** (C++ 特性, 派生类可返回基类返回类型的派生类)

   3. **访问权限不影响重写** (即使基类是 `private virtual`, 派生类也可重写)

2. 匹配规则: 当通过 **基类指针/引用** 调用虚函数时, 编译器根据 **动态类型** 决定调用哪个版本:

   | 场景                       | 调用版本          | 决定时机     |
   | :------------------------- | :---------------- | :----------- |
   | **直接调用** (非指针/引用) | 静态绑定 (编译时) | 对象声明类型 |
   | **基类指针/引用调用**      | 动态绑定 (运行时) | 实际对象类型 |

3. 在 **多重继承 (Multiple Inheritance)** 下, 虚函数的 `override` 规则:
   1. 多重继承中的重写基本规则:
      - **每个基类的虚函数可以独立被重写**: 派生类可以重写来自不同基类的同名虚函数

      - **若多个基类有相同签名的虚函数**: 派生类必须显式重写或解决歧义 (否则编译错误)

      ```cpp
      class Base1 {
      public:
          virtual void foo() { cout << "Base1::foo" << endl; }
      };

      class Base2 {
      public:
          virtual void foo() { cout << "Base2::foo" << endl; }
      };

      class Derived : public Base1, public Base2 {
      public:
          void foo() override { cout << "Derived::foo" << endl; } // 同时重写 Base1 和 Base2 的 foo()
      };

      int main() {
          Derived d;
          Base1* pb1 = &d;
          Base2* pb2 = &d;
          pb1->foo(); // 输出: Derived::foo
          pb2->foo(); // 输出: Derived::foo
          return 0;
      }
      ```

   2. 菱形继承: 当多重继承形成菱形结构时, 需要通过 **虚继承 (Virtual Inheritance)** 避免重复继承, 并明确重写路径

      ```cpp
      class GrandBase {
      public:
          virtual void bar() { cout << "GrandBase::bar" << endl; }
      };

      class Parent1 : public virtual GrandBase {
      public:
          void bar() override { cout << "Parent1::bar" << endl; }
      };

      class Parent2 : public virtual GrandBase {
      public:
          void bar() override { cout << "Parent2::bar" << endl; }
      };

      class Child : public Parent1, public Parent2 {
      public:
          void bar() override { cout << "Child::bar" << endl; } // 必须重写，否则歧义
      };

      int main() {
          Child c;
          GrandBase* pg = &c;
          pg->bar(); // 输出: Child::bar（通过虚继承唯一重写）
          return 0;
      }
      ```

      **关键点**：
      1. 若 `Child` 不重写 `bar()`, 会因 `Parent1` 和 `Parent2` 的重写冲突导致编译错误
      1. **虚继承** 确保 `GrandBase` 在 `Child` 中只有一份副本

   3. 多重继承中的函数隐藏与作用域解析: 如果派生类不重写同名函数, 需通过 **作用域运算符 `::`** 指定调用哪个基类的版本

      ```cpp
      class BaseA {
      public:
          virtual void func() { cout << "BaseA::func" << endl; }
      };

      class BaseB {
      public:
          virtual void func() { cout << "BaseB::func" << endl; }
      };

      class Derived : public BaseA, public BaseB {
      public:
          void callBoth() {
              BaseA::func(); // 调用 BaseA 的版本
              BaseB::func(); // 调用 BaseB 的版本
          }
      };

      int main() {
          Derived d;
          d.callBoth();
          return 0;
      }

      // 输出:
      // BaseA::func
      // BaseB::func
      ```

   4. 虚函数表 (`vtable`) 与多重继承
      - **每个基类有自己的虚表**: 派生类会为每个包含虚函数的基类维护一个虚表指针

      - **派生类重写的函数更新所有相关虚表**: 确保通过任意基类指针调用时都能正确动态绑定

      - **内存布局示意**

        ```text
        Derived 对象:
        +-------------------+
        | Base1 vtable ptr  | -> Derived::foo()
        +-------------------+
        | Base2 vtable ptr  | -> Derived::foo()
        +-------------------+
        | Derived 数据成员   |
        +-------------------+
        ```

4. 使用 `override` 避免错误: 在多重继承中, `override` 关键字能帮助编译器检查重写是否合法:

   ```cpp
   class Derived : public Base1, public Base2 {
   public:
       void foo() override {  // 若 Base1 或 Base2 无虚函数 foo()，编译报错
           cout << "Derived::foo" << endl;
       }
   };
   ```

#### 重载, 重写 (`override`) 和隐藏的区别

- 重载: 在同一范围定义中的同名成员函数才存在重载关系
  主要特点: 函数名相同, 参数类型和数目有所不同, 不能出现参数个数和类型均相同, 仅仅依靠返回值不同来区分的函数
- `override`: 在派生类中覆盖基类中的同名函数, 重写就是重写函数体, 要求基类函数必须是虚函数且:
  - 与基类的虚函数有相同的参数个数
  - 与基类的虚函数有相同的参数类型
  - 与基类的虚函数有相同的返回值类型
- 隐藏: 未声明 `virtual` 关键字的同名派生类函数会屏蔽了基类中的同名函数:
  - 两个函数参数相同, 但是基类函数不是虚函数. **和重写的区别在于基类函数是否是虚函数**
  - 两个函数参数不同, 无论基类函数是不是虚函数, 都会被隐藏. 和重载的区别在于两个函数不在同一个类中.

## 构造和析构函数

## 兼容性原则 or 里氏替换原则 (LSP, Liskov Substitution Principle)

- 派生类对象可以直接赋值给基类对象 (向上转型, Upcasting)
- 基类指针/引用可以指向派生类对象 (多态的基础)
- 三种表现形式:
  - 派生类对象可以赋值给基类对象

    ```cpp
    class Base { /* ... */ };
    class Derived: public Base { /* ... */ };
    Base b;
    Derived d;
    b = d;  // ✅ 合法, 派生类对象赋值给基类对象 (发生"切片")
    ```

    - 注意: 这里会发生对象切片 (Object Slicing), 即 `Base` 只保留 `Derived` 中继承自 `Base` 的部分, `Derived` 特有的成员被丢弃

  - 基类指针可以指向派生类对象

    ```cpp
    Base* pb = &d;  // ✅ 合法, 基类指针指向派生类对象
    ```

  - 基类引用可以绑定派生类对象

    ```cpp
    Base& rb = d;  // ✅ 合法, 基类引用绑定派生类对象
    ```

## 拷贝构造函数

调用的情况:

- 用类的一个实例化对象去初始化另一个对象的时候
- 函数的参数是类的对象时 (非引用传递)
- 函数的返回值是函数体内局部对象的类的对象时, 此时虽然发生 (Named Return Value) NRV 优化, 但是由于返回方式是值传递, 所以会在返回值的地方调用拷贝构造函数

## 类成员初始化方式

- 赋值初始化, 通过在函数体内进行赋值初始化

- 列表初始化, 在冒号后使用初始化列表进行初始化

- 这两种方式的主要区别在于:
  1. 对于在函数体中初始化, 是在所有的数据成员被分配内存空间后才进行的
  1. 列表初始化是给数据成员分配内存空间时就进行初始化, 就是说分配一个数据成员只要冒号后有此数据成员的赋值表达式 (此表达式必须是括号赋值表达式), 那么分配了内存空间后在进入函数体之前给数据成员赋值, 就是说初始化这个数据成员此时**函数体还未执行**.

- 方法一是在构造函数当中做赋值的操作, 而方法二是做纯粹的初始化操作; 我们都知道, C++ 的赋值操作是会产生临时对象的. 临时对象的出现会降低程序的效率

- 必须用到成员列表初始化的情况:
  1. 初始化一个引用成员时
  1. 初始化一个常量成员时
  1. 调用一个基类的构造函数，而它拥有一组参数时
  1. 调用一个成员类的构造函数，而它拥有一组参数时

- 成员初始化列表的作用:
  1. 编译器会一一操作初始化列表, 以适当的顺序在构造函数之内安插初始化操作, 并且在任何显示用户代码之前
  1. list 中的项目顺序是由类中的成员声明顺序决定的, 不是由初始化列表的顺序决定的

### 派生类构造函数

- 执行顺序:
  1. 虚拟基类的构造函数 (多个虚拟基类则按照继承的顺序执行构造函数)
  2. 基类的构造函数 (多个普通基类也按照继承的顺序执行构造函数)
  3. 类类型的成员对象的构造函数 (按照成员对象在类中的定义顺序)
  4. 派生类自己的构造函数
