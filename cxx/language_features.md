## 指针 vs 引用（语义差异）

- **基本语义**：
  - 指针（`T*`）是对象，存的是地址；可以为 `nullptr`，也可以重新指向别处。
  - 引用（`T&`）是“别名语义”：必须在初始化时绑定到某个对象；之后**不能重新绑定**到别的对象。
- **传参语义**：
  - 值传递：会拷贝（或移动）实参到形参。
  - 指针传递：拷贝的是“地址值”；在函数内通过 `*p` 修改可影响原对象，但把形参指针本身改指向不会影响调用方的指针变量。
  - 引用传递：形参与实参指向同一对象；在函数内修改引用等价于修改原对象。
- **实现层面（非标准保证）**：编译器常把引用实现成“隐藏指针”，但这是实现细节；优化后引用可能完全不落内存。
- **生命周期抓手**：
  - 不要返回指向局部变量的指针/引用（会变成悬空）。优先“按值返回”（NRVO/移动优化通常很强），或返回 `std::unique_ptr` 等明确所有权的类型。
  - `const T&` 可以绑定临时对象并触发**延长生命周期**；非常量左值引用 `T&` 不能绑定临时对象。
  - 进阶：`T&&`（右值引用）与“引用折叠/完美转发”见 `modern/c++11.md`。

### 从汇编层去理解“引用像指针”（仅作直觉）

```nasm
9:     int x = 1;
00401048  mov     dword ptr [ebp-4],1

10:    int &b = x;
0040104F  lea     eax,[ebp-4]
00401052  mov     dword ptr [ebp-8],eax
```

- `lea eax,[ebp-4]`：取 `x` 的地址放入寄存器。
- `mov [ebp-8],eax`：把地址保存到某个槽位（很多编译器会用这个槽位来实现“引用别名”）。
- 重要前提：这只是“某次编译结果的实现方式”，不要把它当作语言层的硬规则。

## 声明阅读：括号优先 + 从右往左读

### 分辨以下声明的类型

```c++
int *p[10];      // 指针数组：p 是数组；元素类型是 int*
int (*p)[10];    // 数组指针：p 是指针；指向一个大小为 10 的 int 数组
int *p(int);     // 函数声明：p 是函数；参数 int；返回 int*
int (*p)(int);   // 函数指针：p 是指针；指向“入参 int、返回 int”的函数
```

### 常见 const 组合（辅助读声明）

```c++
const int* p1;   // 指向 const int 的指针（p1 可改指向；*p1 不能改）
int* const p2;   // const 指针（p2 不能改指向；*p2 可以改）
const int* const p3; // const 指针，且指向 const int
```

## 指针安全与术语（推荐放到 memory.md 统一看）

- **空指针**：值为 `nullptr`（或 C 的 `NULL/0`）的指针，不指向任何对象。
- **野指针**：未初始化的指针（值不确定）。
- **悬空指针/引用**：原对象已析构/释放/失效，但指针/引用仍在使用。
- 详见：`memory.md` 的“常见坑（工程视角）”。
